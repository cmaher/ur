# ur_rpc (Shared RPC Contract)

Shared protobuf/tonic definitions between all crates. Changes here affect `ur`, `ur-server`, and `workercmd` binaries.

- Proto files live in `proto/` and are compiled by `build.rs` using `tonic-build`
- All proto modules compile unconditionally. Only `stream` (optional tokio/tracing deps), `error` (optional tonic-types/anyhow deps), and `temporal` (unreleased service) are feature-gated
- All RPC types are generated by prost/tonic from `.proto` files — no hand-written request/response types
